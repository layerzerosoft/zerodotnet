# LayerZero Relational Migrations

LayerZero relational migrations are code-first, forward-only, explicit, and
source-generated by default.

## Packages

- `LayerZero.Migrations`: provider-neutral migration DSL, seed profiles,
  validation, planning, scripting, apply, and baseline flows.
- `LayerZero.Migrations.SqlServer`: SQL Server adapter with SQL rendering,
  schema history storage, `sp_getapplock` coordination, and execution.

## Authoring

Migrations inherit from `Migration` and seeds inherit from `Seed`. Both use
sortable UTC timestamp ids.

```csharp
public sealed class CreateAccountsMigration : Migration
{
    public CreateAccountsMigration()
        : base("20260414120000", "Create accounts")
    {
    }

    public override void Build(MigrationBuilder builder)
    {
        builder.CreateTable("accounts", table =>
        {
            table.Column("id").AsGuid().NotNull();
            table.Column("email").AsString(256).NotNull();
            table.PrimaryKey("id");
        });
    }
}

public sealed class BaselineRolesSeed : Seed
{
    public BaselineRolesSeed()
        : base("20260414121000", "Baseline roles")
    {
    }

    public override void Build(SeedBuilder builder)
    {
        builder.UpsertData("roles", ["id"], row => row.Set("id", 1).Set("name", "admin"));
    }
}
```

Profiles are first-class. `baseline` is the reserved safe default profile.
Additional profiles such as `dev`, `demo`, or `test` are opt-in.

## Registration

Add the runtime, the SQL Server adapter, and the source-generated registry:

```csharp
builder.Services.AddLayerZeroMigrations(options =>
{
    options.Executor = "orders-deploy";
});

builder.Services.AddSqlServer(options =>
{
    options.ConnectionString = connectionString;
});

builder.Services.AddMigrations();
```

`AddMigrations()` is generated at compile time by `LayerZero.Generators`. There
is no runtime assembly scanning in the default path.

## Runtime

`IMigrationRuntime` exposes:

- `InfoAsync`: reports local and applied artifact status.
- `ValidateAsync`: checks checksum drift, missing local definitions, and
  out-of-order additions.
- `ScriptAsync`: produces reviewed SQL for `apply` or `baseline`.
- `ApplyAsync`: executes pending migrations and selected seed profiles.
- `BaselineAsync`: journals selected artifacts without executing them.

`ApplyAsync` refuses to take over a non-empty database without LayerZero
history. Use `BaselineAsync` first for existing databases.

## SQL Server

The SQL Server adapter defaults to:

- one schema history table
- transaction-per-migration execution by default
- non-transactional opt-out per migration
- `SET XACT_ABORT ON`
- `sp_getapplock` runner serialization
- SQL Server-safe identifier quoting and literal rendering

The reference runner host lives in `eng/LayerZero.Migrations.Runner` and shows
the recommended explicit CI/CD execution path:

```bash
dotnet run --project eng/LayerZero.Migrations.Runner -- info --connection-string "<sql-server-connection-string>"
dotnet run --project eng/LayerZero.Migrations.Runner -- validate --connection-string "<sql-server-connection-string>"
dotnet run --project eng/LayerZero.Migrations.Runner -- script --script-kind apply --connection-string "<sql-server-connection-string>" --profile dev
dotnet run --project eng/LayerZero.Migrations.Runner -- apply --connection-string "<sql-server-connection-string>" --profile dev
dotnet run --project eng/LayerZero.Migrations.Runner -- baseline --connection-string "<sql-server-connection-string>"
```
